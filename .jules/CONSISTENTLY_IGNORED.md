## IGNORE: Committing Agent State Files

**- Pattern:** Committing individual agent learning logs or execution journals to the repository (e.g., `.jules/janitor.md`, `.jules/sentinel.md`, `.jules/AGENTS.md`).
**- Justification:** Individual agent state files are transient and should not be committed to the project history.
**- Files Affected:** `.jules/*.md`

## IGNORE: Modifying F-Droid Artifacts Script

**- Pattern:** Refactoring, hardening, moving, or applying formatting fixes to `fdroid/repo/symlinkify_apks`.
**- Justification:** The `fdroid/repo/` directory and its contents are F-Droid artifacts. They should be left as is and not be subject to standard script linting, strict execution rules, or relocation.
**- Files Affected:** `fdroid/repo/symlinkify_apks`

## IGNORE: Mocking CI Tasks

**- Pattern:** Creating a `mise.toml` configuration with placeholder or dummy tasks (e.g., `echo 'Tests passed'`).
**- Justification:** `mise.toml` is intentionally absent and should not be added just to provide mock executions to satisfy automated workflows.
**- Files Affected:** `mise.toml`

## IGNORE: Committing Forbidden Tooling Artifacts

**- Pattern:** Committing files generated by `trunk init`, such as `.trunk/.gitignore` or `.trunk/configs/*`.
**- Justification:** These are local bootstrapping artifacts for linting/formatting and are considered forbidden for version control.
**- Files Affected:** `.trunk/**`
