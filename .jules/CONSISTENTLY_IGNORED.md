## IGNORE: Committing Agent State Files

**- Pattern:** Committing individual agent learning logs or execution journals to the repository (e.g., `.jules/janitor.md`, `.jules/sentinel.md`).
**- Justification:** Individual agent state files are transient and should not be committed to the project history.
**- Files Affected:** `.jules/*.md`

## IGNORE: Modifying F-Droid Artifacts Script

**- Pattern:** Refactoring, hardening, moving, or applying formatting fixes to `fdroid/repo/symlinkify_apks`.
**- Justification:** The `fdroid/repo/` directory and its contents are F-Droid artifacts. They should be left as is and not be subject to standard script linting, strict execution rules, or relocation.
**- Files Affected:** `fdroid/repo/symlinkify_apks`

## IGNORE: Mocking CI Tasks

**- Pattern:** Creating a `mise.toml` configuration with placeholder or dummy tasks (e.g., `echo 'Tests passed'`).
**- Justification:** `mise.toml` is intentionally absent and should not be added just to provide mock executions to satisfy automated workflows.
**- Files Affected:** `mise.toml`

## IGNORE: Committing Forbidden Tooling Artifacts

**- Pattern:** Committing files generated by `trunk init`, such as `.trunk/.gitignore` or `.trunk/configs/*` or `trunk` launcher script.
**- Justification:** These are local bootstrapping artifacts for linting/formatting and are considered forbidden for version control.
**- Files Affected:** `.trunk/**`, `trunk`

## IGNORE: Fixing Markdown Lint Issues in README

**- Pattern:** Modifying `README.md` to fix markdown lint issues (e.g., wrapping bare URLs in angle brackets).
**- Justification:** These changes have been consistently rejected; `README.md` should be left as is despite markdown lint drift.
**- Files Affected:** `README.md`

## IGNORE: Creating AGENTS.md Conventions File

**- Pattern:** Creating an `AGENTS.md` file in the root directory to store repository conventions.
**- Justification:** Although it is the default global instruction to create `AGENTS.md` when it does not exist, doing so has been consistently rejected in this repository and should be avoided.
**- Files Affected:** `AGENTS.md`

## IGNORE: Adding Centralized Error Reporting Script

**- Pattern:** Adding or sourcing a centralized error-reporting utility (e.g., `src/utils/error_reporting.sh`).
**- Justification:** Attempts to refactor scripts to use a centralized error handler have been consistently rejected.
**- Files Affected:** `src/utils/error_reporting.sh`
